# 破局层规则

> 功能：挖掘问题根本原因，打破固有假设，找到突破口

---

## 调用规则

```
破局层必须按以下顺序调用思维模型：

┌─────────────────────────────────────────────────────────────┐
│ Step 1: MECE原则（必调）                                     │
│         先用MECE确保问题拆解完整无遗漏                        │
│         文件：models/mece.md                                │
├─────────────────────────────────────────────────────────────┤
│ Step 2: 5Why分析法（必调）                                   │
│         对MECE拆解的每个子问题追问根本原因                    │
│         文件：models/5why.md                                │
├─────────────────────────────────────────────────────────────┤
│ Step 3: 第一性原理 或 矛盾分析法（二选一）                    │
│                                                             │
│         创新设计/成本重构场景 → 第一性原理                    │
│         文件：models/first-principles.md                    │
│                                                             │
│         危机处理/战略选择/复杂决策场景 → 矛盾分析法            │
│         文件：models/contradiction-analysis.md              │
└─────────────────────────────────────────────────────────────┘
```

---

## 问题类型与模型选择

| 问题类型 | Step 3 选择 | 选择理由 |
|---------|------------|---------|
| 危机排查 | 矛盾分析法 | 需要找到主要矛盾，集中力量解决 |
| 创新设计 | 第一性原理 | 需要打破常规，从基本事实重构 |
| 复杂决策 | 矛盾分析法 | 需要处理多方利益博弈 |
| 效能提升 | 第一性原理 | 需要重新审视流程本质 |

---

## 输出格式

```
═══════════════════════════════════════════════════════════════
                    🔓 破局层分析
═══════════════════════════════════════════════════════════════

【MECE拆解】

问题：[描述问题]

分解维度：[时间/空间/结构/过程/要素]

分解结构：
├── [类别A]
│   ├── [A-1]
│   └── [A-2]
└── [类别B]
    ├── [B-1]
    └── [B-2]

MECE检验：
• 相互独立：[✓/✗] — [各部分是否有重叠？]
• 完全穷尽：[✓/✗] — [是否有遗漏？]

───────────────────────────────────────────────────────────────

【5Why根因分析】

针对关键子问题：[哪个子问题最关键？]

Why 1: 为什么[问题]？
→ [原因1]

Why 2: 为什么[原因1]？
→ [原因2]

Why 3: 为什么[原因2]？
→ [原因3]

Why 4: 为什么[原因3]？
→ [原因4]

Why 5: 为什么[原因4]？
→ [根本原因]

三个层面检查：
┌───────────────────┬────────────────────────────────────────┐
│ 层面              │ 分析                                   │
├───────────────────┼────────────────────────────────────────┤
│ 为什么会发生？    │ [从制造/发生角度分析]                   │
│ 为什么没发现？    │ [从检验/检测角度分析]                   │
│ 为什么没预防？    │ [从系统/流程角度分析]                   │
└───────────────────┴────────────────────────────────────────┘

───────────────────────────────────────────────────────────────

【第一性原理/矛盾分析法】

[根据问题类型选择对应的模型进行分析]

[如果选择第一性原理]：
传统假设：
1. [假设1] — 基于[惯例/经验]
2. [假设2] — 基于[惯例/经验]

基本事实拆解：
• [组成部分A]：[物理事实/市场价格]
• [组成部分B]：[物理事实/市场价格]

重构方案：
→ 基于[基本事实]，新方案是...

[如果选择矛盾分析法]：
存在的矛盾：
1. [矛盾A]
2. [矛盾B]
3. [矛盾C]

主要矛盾：[哪个最关键？]
矛盾方面：主要方面[...]，次要方面[...]

行动策略：
→ 集中力量解决[主要矛盾]

───────────────────────────────────────────────────────────────

破局层洞察：
→ [一句话总结最关键的发现]
```
