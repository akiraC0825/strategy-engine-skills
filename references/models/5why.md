# 5Why分析法

> 破局层模型 | 用于阶段二：深度拆解与推演

---

## 核心定义

通过连续追问"为什么"来找到问题的根本原因，而非停留在表面症状。5Why分析法（又称"5问法"）是一种诊断性技术，用来识别和说明因果关系链，其根源会引起恰当地定义问题。

### 大野耐一与5Why的诞生

> "我碰到问题至少要问5个为什么。" —— 大野耐一

5Why首创自丰田公司的大野耐一，源于一次新闻发布会。有人问："丰田公司的汽车质量怎么会这么好？"他回答："我碰到问题至少要问5个为什么。"

大野耐一总是爱在车间走来走去，停下来向工人发问。他反复地就一个问题，问"为什么"，直到回答令他满意，被他问到的人也心里明白为止——这就是著名的"五个为什么"的起源。

### 核心著作

- **《丰田生产方式》**：大野耐一著，阐述5Why在质量管理中的应用

## 核心概念

### 5Why的本质

```
┌─────────────────────────────────────────────────────────┐
│ 不是问5次，而是问到找到根本原因为止                      │
│                                                         │
│ 有时3次就够了                                           │
│ 有时需要10次                                            │
│ 关键是"打破砂锅问到底"                                   │
└─────────────────────────────────────────────────────────┘
```

### 三个分析层面

| 层面 | 角度 | 核心问题 |
|------|------|---------|
| 为什么会发生？ | 制造角度 | 问题产生的直接原因 |
| 为什么没有被发现？ | 检验角度 | 检测机制的漏洞 |
| 为什么系统没有预防？ | 体系角度 | 流程设计的缺陷 |

### 三现主义

5Why分析法需要时刻围绕"三现主义"来聚焦思路和启发灵感：

| 原则 | 含义 | 实践要求 |
|------|------|---------|
| **现实** | 基于现实情况分析原因 | 而不是主观臆断，用数据说话 |
| **现场** | 亲自到问题出现的一线 | 不在办公室猜测，要去现场观察 |
| **现物** | 亲自看实物、接触实物 | 不依赖报告，要亲眼确认 |

> "经常去现场观察，才能发现应该做什么，应该改善什么。" —— 大野耐一

## 核心方法论

### 实施步骤

```
第一部分：把握现状

Step 1: 识别问题
        了解可能大、模糊或复杂的问题

Step 2: 澄清问题
        明确实际发生了什么以及应该发生什么

Step 3: 分解问题
        将问题分解为小的、独立的元素

Step 4: 查找原因要点(PoC)
        追溯第一手的原因要点

Step 5: 把握问题的倾向
        问：谁？哪个？什么时间？多少频次？多大量？

第二部分：原因调查

Step 6: 识别并确认直接原因
        如果原因可见，验证；如果不可见，考虑潜在原因

Step 7: 使用5Why建立原因/效果关系链
        连续问为什么，直到找到根本原因
```

### 关键原则

```
┌─────────────────────────────────────────────────────────┐
│ ✓ 每次只问一个"为什么"                                  │
│ ✓ 答案必须基于事实，而非推测                            │
│ ✓ 关注"为什么"而非"谁"（不追究责任）                   │
│ ✓ 找到可控的根本原因                                    │
│ ✓ 鼓励解决问题的人避开主观或自负的假设和逻辑陷阱        │
│ ✓ 从结果着手，沿着因果关系链条，顺藤摸瓜                │
│ ✗ 不要把5Why变成推卸责任的工具                          │
│ ✗ 不要在没有验证的情况下假设因果关系                    │
└─────────────────────────────────────────────────────────┘
```

### 判断根本原因的标准

当出现以下情况时，可以认为找到了根本原因：

1. **可操作性**：原因是可以被控制和改变的
2. **系统性**：原因来自系统或流程，而非个人
3. **可验证性**：原因可以通过数据或观察来验证
4. **可预防性**：解决该原因可以防止问题再次发生

## 经典案例

### 案例一：机器停机分析（大野耐一经典案例）

这是5Why分析法最经典的案例，直接来自丰田生产现场：

```
问题：机器停机了

Why 1：为什么机器停机了？
→ 因为超负荷，保险丝断了

Why 2：为什么超负荷？
→ 因为轴承润滑不足

Why 3：为什么润滑不足？
→ 因为润滑泵吸不上油

Why 4：为什么吸不上油？
→ 因为油泵轴磨损、松动了

Why 5：为什么油泵轴磨损了？
→ 因为没有安装过滤器，混进了铁屑等杂质

根本原因：油泵没有安装过滤器
解决方案：在油泵上安装过滤器（而非简单更换保险丝）
```

**案例启示**：如果没有这种追根究底的精神来发掘问题，很可能只是换根保险丝草草了事，真正的问题还是没有解决。

### 案例二：杰弗逊纪念大厦墙面裂纹

```
问题：华盛顿广场杰弗逊纪念大厦墙面出现裂纹

Why 1：为什么墙面出现裂纹？
→ 因为每天用酸性清洁剂冲洗墙壁

Why 2：为什么要每天冲洗墙壁？
→ 因为墙面上每天都有大量鸟粪

Why 3：为什么有鸟粪？
→ 因为大厦周围聚集了很多燕子

Why 4：为什么有燕子？
→ 因为墙面上有很多蜘蛛，燕子喜欢吃蜘蛛

Why 5：为什么有蜘蛛？
→ 因为墙面上有很多虫子，蜘蛛喜欢吃虫子

Why 6：为什么有虫子？
→ 因为大厦窗户长期透光，虫子有趋光性

根本原因：窗户长期透光吸引虫子
解决方案：拉上窗帘（而非更换墙面材料或增加清洁频次）
```

**案例启示**：通过追根溯源，发现了一个完全意想不到的解决方案——拉窗帘，而不是最初假设的更换墙面材料或减少清洁次数。

### 案例三：车间地板油渍分析

```
问题：车间地板上有油渍

Why 1：为什么地板上有油？
→ 因为机器漏油

Why 2：为什么机器会漏油？
→ 因为油箱密封圈老化

Why 3：为什么密封圈会老化？
→ 因为采购的密封圈质量不合格

Why 4：为什么采购了不合格的密封圈？
→ 因为供应商评估标准中没有密封件的质量要求

Why 5：为什么没有质量要求？
→ 因为供应商管理制度缺失关键零部件的质量标准

根本原因：供应商管理制度缺失关键零部件质量标准
解决方案：完善供应商管理制度，建立关键零部件质量标准
```

### 案例四：生产效率低下分析

```
问题：某生产线生产效率比目标低20%

Why 1：为什么效率低？
→ 因为经常停线

Why 2：为什么经常停线？
→ 因为零件供应不及时

Why 3：为什么零件供应不及时？
→ 因为仓库经常缺料

Why 4：为什么仓库经常缺料？
→ 因为采购计划不准确

Why 5：为什么采购计划不准确？
→ 因为销售预测数据没有及时共享给采购部门

根本原因：销售预测信息共享机制缺失
解决方案：建立销售与采购的信息共享平台和流程
```

## 丰田生产方式思维框架

### 丰田生产方式(TPS)与5Why的关系

5Why分析法是丰田生产方式（Toyota Production System, TPS）的核心工具之一，与TPS的其他核心理念紧密相连：

```
┌─────────────────────────────────────────────────────────┐
│                  丰田生产方式(TPS)核心支柱              │

│   │ 准时化生产  │      │  自动化     │                 │
│   │    (JIT)    │      │ (Jidoka)    │                 │
│   └──────┬──────┘      └──────┬──────┘                 │
│          │                    │                        │
│          └────────┬───────────┘                        │
│                   ↓                                    │
│         ┌─────────────────┐                            │
│         │   持续改善      │                            │
│         │   (Kaizen)      │                            │
│         └────────┬────────┘                            │
│                  ↓                                     │
│         ┌─────────────────┐                            │
│         │  5Why分析法     │  ← 核心工具                │

│         └─────────────────┘                            │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 丰田思维的五大原则

| 原则 | 含义 | 与5Why的关系 |
|------|------|-------------|
| **以问题为出发点** | 注重发现和解决问题 | 5Why是发现问题的核心方法 |
| **价值流分析** | 识别价值链中的浪费 | 用5Why分析浪费产生的原因 |
| **精益生产** | 消除浪费、提高效率 | 5Why帮助找到浪费的根源 |
| **持续改进** | 每天解决问题推动成长 | 5Why是持续改进的工具 |
| **快速反应能力** | 迅速适应变化和需求 | 5Why加速问题的解决 |

### 丰田式问题解决流程

```
┌──────────────────────────────────────────────────────────┐
│           丰田式问题解决流程(TPS Problem Solving)         │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  Step 1: 识别问题                                        │
│          ↓ (现地现物：亲自到现场观察)                    │
│                                                          │
│  Step 2: 分析问题                                        │
│          ↓ (使用5Why深入分析根本原因)                    │
│                                                          │
│  Step 3: 制定对策                                        │
│          ↓ (针对根本原因，而非表面症状)                  │
│                                                          │
│  Step 4: 实施对策                                        │
│          ↓ (在实际工作中实施改进措施)                    │
│                                                          │
│  Step 5: 验证效果                                        │
│          ↓ (监控效果，确保问题不再发生)                  │
│                                                          │
│  Step 6: 标准化                                          │
│          (将有效措施纳入标准作业程序)                    │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

### 5S方法与5Why的协同

丰田的5S方法为5Why分析提供了良好的现场基础：

| 5S要素 | 日语 | 含义 | 对5Why分析的贡献 |
|--------|------|------|-----------------|
| 整理 | Seiri | 清除不必要的物品 | 减少干扰，聚焦问题 |
| 整顿 | Seiton | 物品有序摆放 | 便于发现异常 |
| 清扫 | Seiso | 保持清洁 | 使问题更容易暴露 |
| 清洁 | Seiketsu | 标准化前三步 | 建立可比较的基准 |
| 素养 | Shitsuke | 培养良好习惯 | 形成问题意识 |

## 常见陷阱

| 陷阱 | 表现 | 避免方法 |
|------|------|---------|
| 答案跳跃 | 从表面直接跳到根本原因，中间缺少逻辑链 | 确保每一步都有因果关系 |
| 归咎于人 | 把原因归结为"某人犯错" | 关注系统和流程，而非个人 |
| 过早停止 | 找到表面原因就停止 | 继续追问直到找到可控的根本原因 |
| 无限循环 | 追问到无法行动的哲学层面 | 停在可操作的系统/流程层面 |
| 主观臆断 | 凭想象推断原因 | 坚持"三现主义"，基于事实和数据 |
| 线性思维 | 忽略一个问题的多个可能原因 | 结合鱼骨图，考虑多因素 |
| 责任逃避 | 用5Why来推卸责任 | 强调"为什么"而非"谁" |

## 延伸阅读

### 推荐书籍

1. **《丰田生产方式》** - 大野耐一
   - 5Why分析法的原始出处
   - 理解丰田生产方式的核心理念

2. **《改变世界的机器》** - James P. Womack
   - 精益生产的经典著作
   - 理解5Why在精益思想中的位置

3. **《学习型组织》** - Peter Senge
   - 系统思考与5Why的结合
   - 理解组织学习与问题解决的关系

### 实践建议

1. **从简单问题开始**：初学者应从简单、单一原因链的问题开始练习
2. **坚持三现主义**：不到现场，不摸清事实，不要轻易下结论
3. **团队协作**：邀请了解现场情况的人员参与分析
4. **验证每一步**：每个"为什么"的答案都要用事实验证
5. **记录和分享**：将分析过程和结果记录下来，形成组织知识

## 总结

5Why分析法看似简单——只是连续问"为什么"，但要真正用好它，需要：

1. **正确的心态**：追求真相，不追责
2. **科学的方法**：基于事实，不臆断
3. **系统的思维**：看到整体，不孤立
4. **持续的行动**：解决问题，不停留

> "通过重复五次提问，问题的本质及解决办法便会清晰呈现。" —— 大野耐一

5Why不仅是一种分析工具，更是一种追求卓越、持续改进的精神。
