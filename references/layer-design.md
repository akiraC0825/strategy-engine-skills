# 构思层规则

> 功能：生成创新方案，多角度审视问题，找到最优解

---

## 调用规则

```
构思层按以下规则调用思维模型：

┌─────────────────────────────────────────────────────────────┐
│ Step 1: 六顶思考帽（必调）                                   │
│         多角度审视破局层的洞察和初步方案                      │
│         文件：models/six-thinking-hats.md                   │
├─────────────────────────────────────────────────────────────┤
│ Step 2: 根据问题类型选择补充模型（选调）                      │
│                                                             │
│         创新设计场景：                                       │
│         • 正反合辩证法 → 处理方案的对立统一                   │
│         • 文件：models/thesis-antithesis-synthesis.md       │
│                                                             │
│         复杂决策/投资场景：                                  │
│         • 第二层思维 → 找到与市场共识的差异                   │
│         • 文件：models/second-level-thinking.md             │
│                                                             │
│         商业分析/竞争分析场景：                              │
│         • 多元思维模型 → 跨学科视角分析                       │
│         • 文件：models/latticework.md                       │
└─────────────────────────────────────────────────────────────┘
```

---

## 问题类型与模型选择

| 问题类型 | Step 2 选择 | 选择理由 |
|---------|------------|---------|
| 创新设计 | 正反合辩证法 | 方案需要迭代优化，处理对立意见 |
| 复杂决策 | 第二层思维 | 需要找到与大众共识的差异化判断 |
| 投资决策 | 第二层思维 + 多元思维 | 需要逆向思考和多学科视角 |
| 效能提升 | 多元思维模型 | 需要系统优化，多学科视角 |
| 危机排查 | 无需补充 | 六顶思考帽已足够 |

## 输出格式

```
═══════════════════════════════════════════════════════════════
                    💡 构思层分析
═══════════════════════════════════════════════════════════════

【六顶思考帽】

⚪ 白帽（事实与信息）：
→ 已知事实：[...]
→ 需要信息：[...]
→ 数据支撑：[...]

🔴 红帽（情感与直觉）：
→ 第一感受：[...]
→ 直觉判断：[...]
→ 担忧与期待：[...]

🟡 黄帽（价值与收益）：
→ 好处是什么：[...]
→ 为什么可行：[...]
→ 潜在收益：[...]

⚫ 黑帽（风险与批判）：
→ 可能的问题：[...]
→ 最大的风险：[...]
→ 为什么会失败：[...]

🟢 绿帽（创造与可能性）：
→ 替代方案：[...]
→ 如何改进：[...]
→ 创新想法：[...]

🔵 蓝帽（总结与决策）：
→ 关键发现：[...]
→ 行动方向：[...]

───────────────────────────────────────────────────────────────

【补充模型分析】（根据问题类型调用）

[如果选择正反合辩证法]：
正题（当前方案）：
→ [...]

反题（对立观点）：
→ [...]

合题（扬弃整合）：
→ 保留：[从正题保留什么]
→ 吸收：[从反题吸收什么]
→ 抛弃：[抛弃哪些不合理部分]
→ 新方案：[...]

[如果选择第二层思维]：
第一层思维（市场共识）：
→ 大多数人认为：[...]

第二层思维（我的差异）：
→ 我认为：[...]
→ 与共识的差异：[...]

验证逻辑：
→ 支持我判断的事实：[...]

[如果选择多元思维模型]：
【经济学视角】
→ 供需关系：[...]
→ 成本收益：[...]

【心理学视角】
→ 激励机制：[...]
→ 认知偏差：[...]

【数学视角】
→ 概率评估：[...]

交叉验证：
→ 各学科结论是否一致？[是/否]

───────────────────────────────────────────────────────────────

构思层洞察：
→ [一句话总结最关键的发现或方向]
```
