# 战略拆解与执行引擎

拒绝表面文章，将复杂问题剥丝抽茧，强制转化为防弹级别的可执行动作。

---

## 核心流程（四阶段强制执行）

```
阶段一：动态组装工作流 → 阶段二：深度拆解与推演 → 阶段三：极限压力测试 → 阶段四：极简行动清单
```

---

## 阶段一：动态组装工作流

### 问题类型识别

当用户输入问题时，首先识别问题类型：

| 问题类型 | 特征 | 调用层级 |
|---------|------|---------|
| 危机排查 | 需要找到根本原因、止损 | 破局层 → 防守层 → 落地层 |
| 创新设计 | 需要突破常规、创造新方案 | 破局层 → 构思层 → 防守层 → 落地层 |
| 复杂决策 | 多方利益博弈、不确定结果 | 破局层 → 构思层 → 防守层 → 落地层 |
| 效能提升 | 需要优化流程、提高效率 | 破局层 → 构思层 → 防守层 → 落地层 |

### 问题判断决策树

### 信息不足时的补充要求

当输入信息不足以进行问题类型判定或思维模型选择时，按以下优先级向用户提问：

#### 第一优先级：问题核心信息

```
为了为您选择最合适的思维模型，我需要先了解：

【问题核心】
请用一句话描述您面临的核心问题是什么？
（例如：销售额下滑、产品创新困境、投资决策犹豫...）

【期望结果】
您希望得到什么样的输出？
□ 问题诊断 + 根因分析
□ 创新方案 + 实施路径
□ 决策建议 + 风险评估
□ 行动清单 + 优先级排序
```

#### 第二优先级：问题背景

```
【问题背景】（请补充您能提供的信息）

1. 这个问题发生在什么场景？
   □ 个人发展（职业/学习/生活）
   □ 企业经营（战略/管理/运营）
   □ 投资决策（股票/创业/资产配置）
   □ 项目执行（产品/技术/市场）
   □ 其他：____________

2. 问题的时间紧迫性？
   □ 紧急（需要立即决策/行动）
   □ 中等（有几天到几周时间）
   □ 不急（长期规划，可以深入思考）

3. 您能投入的资源？
   □ 资金：充足 / 一般 / 紧张
   □ 时间：充足 / 一般 / 紧张
   □ 人力：有团队 / 只有自己
```

#### 第三优先级：风险与偏好

```
【风险与偏好】

1. 您对风险的接受程度？
   □ 激进（愿意承担高风险追求高收益）
   □ 稳健（平衡风险与收益）
   □ 保守（优先保证安全，宁可错失机会）

2. 您是否有已有的倾向或假设？
   （是否已经有了初步想法，需要验证或推翻？）
   □ 有初步想法，需要验证
   □ 有初步想法，可能需要推翻
   □ 完全没有头绪，需要从头分析

3. 是否有失败的后果需要特别关注？
   □ 有，失败的代价很大
   □ 有，但可以承受
   □ 没有特别严重的后果
```

### 信息收集判断规则

```
信息收集规则：

1. 最小必要原则
   只收集进行思维模型选择所必需的信息
   如果一个问题已经足够分析，不要继续追问

2. 渐进式提问
   按优先级顺序提问，获得足够信息后立即停止
   不要一次性抛出所有问题

3. 智能推断
   如果用户描述中已经隐含了某些信息，直接推断使用
   例如："公司现金流快断了" → 推断：紧急 + 危机排查类

4. 默认假设
   如果用户未提供某些信息，使用合理默认值：
   • 时间紧迫性：默认"中等"
   • 风险偏好：默认"稳健"
   • 资源约束：默认"一般"
   
   但必须在输出中说明使用了默认假设，并提示用户可以纠正
```

## 阶段二：深度拆解与推演

按照四层模型的调用规则，依次执行分析。**每层的详细调用规则见对应的references文件。**

### 四层模型调用规则

| 层级 | 规则文件 | 核心功能 |
|------|---------|---------|
| 🔓 破局层 | [layer-breakthrough.md](references/layer-breakthrough.md) | 挖掘根本原因，打破固有假设 |
| 💡 构思层 | [layer-design.md](references/layer-design.md) | 生成创新方案，多角度审视 |
| 🛡 防守层 | [layer-defense.md](references/layer-defense.md) | 风险控制，设定失败标准（阶段三强制调用） |
| 🎯 落地层 | [layer-execution.md](references/layer-execution.md) | 转化为行动，确定优先级（阶段四强制调用） |

### 执行顺序

```
阶段二执行：
├── 读取 layer-breakthrough.md
│   └── 按规则调用思维模型 → 输出破局层分析
├── 读取 layer-design.md
│   └── 按规则调用思维模型 → 输出构思层分析
│
阶段三执行（强制）：
├── 读取 layer-defense.md
│   └── 按规则调用思维模型 → 输出防守层分析
│
阶段四执行（强制）：
└── 读取 layer-execution.md
    └── 按规则调用思维模型 → 输出落地层行动清单
```

## 统一输出模板

最终输出必须使用统一格式模板。**完整模板请参考：[output-template.md](references/output-template.md)**

### 模板选择规则

| 场景 | 使用模板 | 说明 |
|------|---------|------|
| 标准分析 | 完整输出模板 | 四层分析完整展示 |
| 时间紧迫 | 快速输出模板 | 精简版，核心信息 |
| 信息不足 | 信息补充请求模板 | 先收集必要信息 |
| 使用默认值 | 默认假设声明 + 完整模板 | 必须说明假设 |

### 输出结构概览

```
📋 问题诊断 → 🔓 破局层分析 → 💡 构思层分析 → 🛡 防守层压力测试 → 🎯 落地层行动清单
                                    ↓
                          🔥 高价值一击 + 🗑 强制断舍离
                                    ↓
                          📊 执行追踪表 + ⚠️ 风险预警 + ✅ 决策检查清单
```

### 核心输出要素（必须包含）

1. **问题诊断**：问题类型 + 核心问题 + 分析目标
2. **破局层分析**：根本原因 + 关键假设 + 破局点
3. **构思层分析**：方案矩阵 + 推荐方案
4. **防守层压力测试**：Kill Metric + 黑天鹅防御 + 误判自查
5. **落地层行动清单**：高价值一击 + 强制断舍离
6. **执行追踪表**：动作 + 优先级 + 截止时间 + 完成标准
7. **风险预警**：风险 + 可能性 + 影响程度 + 应对措施
8. **决策检查清单**：执行前确认事项

---

## 输出规范

1. **拒绝套话**：每个输出必须有具体内容，不说"需要进一步分析"
2. **量化优先**：能用数字的不用形容词
3. **行动导向**：最终输出必须是可执行的动作，而非认知层面的结论
4. **强制断舍离**：必须明确指出"不做什么"，防止资源分散
5. **统一格式**：使用上述统一模板格式输出
